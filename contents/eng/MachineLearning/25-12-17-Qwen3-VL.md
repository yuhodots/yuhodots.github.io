> A quick overview and summary of key points from the Qwen3-VL Technical Report.

### Introduction

Qwen3-VL was designed to address the following problems:

- Catastrophic forgetting where text processing ability degrades while improving vision capabilities
- Context window limitations or degraded positional encoding resolution when handling long videos or large documents

To address these issues, **Qwen3 language model is used as the backbone**, and the model is **optimized so that text performance does not degrade even after visual training**.

- Dense models range from 2B to 32B parameters
- MoE models include 30B A3B and 225B A22B
  - A3B: 3B active parameters
  - A22B: 22B active parameters

### Architecture

![img](../../img/25-12-13-1.png)

While inheriting Qwen2.5-VL's 3-module structure, three key architectural upgrades were made:

1. vision encoder
2. LLM
3. vision-language merger

##### Vision Encoder & Dynamic Resolution

Google's **SigLIP-2** was adopted as the vision encoder. SigLIP is known to provide better semantic understanding and localization capabilities compared to conventional CLIP.

- CLIP is computed using the formula below, calculating cross entropy between all text/image pairs within a batch:

  - Interpretation: Optimized so that the 'relative' similarity between matched image-text pairs within a batch is high. In other words, it only needs to be 'relatively' closer to the correct image/text than other pair images or texts.

  ![img](../../img/25-12-13-2.png)

- SigLIP uses sigmoid instead of softmax

  - It is not asymmetric and does not require a global normalization factor. Each **image-text pair is evaluated independently**. That is, the similarity must be absolute rather than relative. This likely makes it less sensitive to how data batches are composed.

To adapt well to various input image resolutions, **2D-RoPE** and a method of **interpolating absolute position embeddings** based on input size are used. RoPE (Rotary Positional Embedding) is excellently explained [here](https://medium.com/@hugmanskj/mastering-llama-rotary-positional-embedding-rope-이해하기-9b1963a22852).

The 2B and 4B models use SigLIP2-Large for lightweight deployment, while models 8B and above use the SigLIP2-SO-400M variant.

A 2-layer MLP is used as the **MLP-based Vision Language Merger**, which **compresses 2x2 visual features from the vision encoder into a single visual token**.

##### Interleaved MRoPE

MRoPE was first introduced in Qwen2-VL. It works by simply dividing time (t), height (h), and width (w) dimensions and assigning rotation frequencies. The front portion of the embedding vector handles time, the middle handles height, and the rear handles width.

However, the research team discovered that this approach causes an imbalanced frequency spectrum. To address this, they proposed interleaved MRoPE, which ensures that **each dimension (time, h, w) is uniformly distributed across both low-frequency and high-frequency bands**. This allows the model to learn local texture information (high frequency) and global structural information (low frequency) in a balanced manner across all dimensions, significantly improving video understanding and long-context processing performance.

- Imbalanced frequency spectrum: When information for a specific dimension is concentrated only in the high-frequency band or only in the low-frequency band, it hinders learning long-range dependencies in long video sequences. Values in the earlier positions of the embedding vector change rapidly with each fixed angle increment.

- Mechanism: The time (t), height (h), and width (w) components are **interleaved** across the embedding dimensions. For dimension d of the vector, information is encoded in the order [**t, h, w, t, h, w, ...**].

##### **DeepStack**

**Traditional VLMs project only the last layer output of the vision encoder (ViT) and inject it into the LLM** (e.g., LLAVA). However, the last layer of the vision encoder contains only highly abstracted semantic information, potentially losing low-level information needed for tasks like fine-grained object identification in OCR. To address this, the **DeepStack** mechanism was proposed. This approach allows the LLM to leverage rich visual signals from low-level texture information to high-level semantic information, without increasing additional context token length (visual token count is maintained).

- Mechanism: **Visual features are extracted not only from the last layer of the ViT but also from intermediate layers**. The extracted multi-level features pass through separate MLP mergers and are then injected into different layers of the LLM (primarily early layers).

##### Explicit Video Timestamp

Qwen2.5-VL **implicitly processed video temporal information through time-synchronized positional embeddings**. However, this causes position IDs to grow excessively large and become sparse for long videos, and is inefficient when handling data with varying FPS. Qwen3-VL replaces this with **text-based timestamp tokens**.

- Implementation: An explicit **text-string timestamp such as `<3.0 seconds>` is inserted before each temporal patch** of the video.
- Training strategy: The training data is designed to **generate all H, M, S (hours, minutes, seconds) formats** in seconds, enabling the model to understand various time representations as natural language.

While context length increases slightly, the model can directly perceive the flow of time through text, enabling much more accurate answers to temporal grounding questions such as "What happened at 3 minutes and 20 seconds of the video?" This dramatically improves dense captioning and event localization performance.

##### Square-root Reweighting

Qwen3-VL applies **square-root normalized per-token loss**. This **dynamically adjusts the learning contribution between text-only data and multimodal data**, contributing to maximizing multimodal performance without degrading text capability. In other words, the loss value is not used directly but adjusted (reweighted) using square-root to reduce or balance it.

### Pre-training

Pre-training proceeds sequentially in 4 stages to progressively expand the model's capabilities, training on more than 2 trillion tokens of data.

![img](../../img/25-12-13-3.png)

##### Stage 0: Vision-Text Alignment

- Purpose: **Initial alignment to bridge the modality gap between the vision encoder and LLM**
- Method: The vision encoder and LLM backbone are frozen, and only the **MLP-based merger parameters are trained**
- Data: Approximately 67B tokens of high-quality image-caption pairs, visual knowledge data, and OCR data. Sequence length is set to 8192.

##### Stage 1: Multi-modal Pre-training

- Purpose: Broad acquisition of visual knowledge through large-scale data
- Method: **All parameters are set to trainable** for **end-to-end training**
- Data: Approximately 1T (trillion) token dataset. Text-only data is mixed with multimodal data to preserve text capabilities. Sequence length remains at 8192.
  - Multimodal data types: interleave documents, textual grounding, VQA, STEM, etc.

##### Stage 2: Long-Context Pre-training

- Purpose: Building long sequence processing capabilities and video/agent task abilities
- Method: **Sequence length is expanded 4x to 32K (32768) for full parameter training**
- Data: An additional approximately 1T (trillion) tokens of data are trained. The **proportion of text data is increased** for long-text understanding, while for multimodal data, **the proportion of video data and agent instruction-following data is significantly increased**.

##### Stage 3: Ultra-Long-Context Adaptation

- Purpose: Acquiring extreme context length (256K) processing capability
- Method: **Sequence length is expanded to 256K (262144)**
- Data: Approximately 100B (billion) tokens of highly curated data are used, primarily focusing on tasks such as **long video summarization and full book analysis**.

### Training Data

##### Image Caption & Interleaved Data

Simple web-crawled data is noisy and contains poor descriptions.

- **Recaptioning**: Fine-tuned Qwen2.5-VL-32B model to regenerate large-scale image caption datasets. Descriptions are enriched to cover not only key objects but also attributes, spatial arrangements, and contextual meaning.
- **Interleaved data**: Large volumes of **document data where text and images appear together**, such as papers, magazines, and textbooks, were collected. Preprocessing includes merging while maintaining page order into 256K token sequences, with domain classifiers and score models used to filter ads and low-quality content.

##### Knowledge & OCR

- **Entity-based sampling**: Entities are defined across 10+ categories including animals, plants, landmarks, and products, with importance-based sampling performed to cover long-tailed entities.
- **Multilingual OCR**: OCR capability expanded from 10 to 39 languages by adding 29 new languages. This was achieved using 30 million internally collected images and synthetic data.

##### Grounding & Counting

- Large volumes of grounding data based not only on **bounding boxes** but also **points** were produced.
- Using specialized models like **Grounding DINO** and Qwen2.5-VL, a pipeline was built to automatically generate and verify high-quality object location annotations for unlabeled images. Coordinates are normalized for robustness against resolution changes.

##### Spatial Understanding & 3D Recognition

Additional data was included for 3D spatial understanding:

- **Spatial relationships**: Datasets for learning relative positional relationships such as "the book behind the laptop" or "the cup on the left" were constructed.
- **3D grounding**: Data for estimating 3D positions of objects from monocular images was included. The Omni3D dataset was unified to a virtual camera coordinate system to overcome differences across various sensor sources.

##### Code & Agent

- **Multimodal Coding**: Multimodal coding data was significantly enhanced, including converting UI screenshots to HTML/CSS and implementing diagrams as Python code.
- **GUI Agent**: Trajectory data was collected from desktop, mobile, and web environments. Beyond simple clicks, CoT data including planning and self-correction processes was synthesized to enhance agent intelligence.

##### Video

- **Dense Caption Synthesis**: Story-level captions with timestamps were generated for long videos. A strategy was used where descriptions for short clips are generated first, then merged sequentially to create coherent long descriptions.
- **Length adaptive sampling**: Frame counts and FPS were dynamically adjusted per training stage, enabling the model to learn both the details of short videos and the overall context of long videos.

##### STEM & Math

- Synthetic diagrams: Geometry shapes, function graphs, etc. were rendered using code, and questions about them (intersections, angles, etc.) were generated to enhance visual precision.
- Reasoning data: Over 60 million K-12 and university-level math/science problems were curated. A standardized CoT format with step-by-step solutions was adopted to cultivate the model's logical reasoning ability.

### Post-training

Post-training is the process of aligning and specializing knowledge to user intent. At this stage, Qwen3-VL is split into two model modes: Thinking and Non-Thinking. Approximately 1.2 million high-quality instruction tuning samples were used.

- Thinking model: Intensively trained on **long CoT data that explicitly generates reasoning processes**. Problems solvable without visual information are removed to maximize multimodal reasoning.
- Non-Thinking model: Uses **standard QA format** data.

After initial training on 32K length data, a second round of training is conducted by mixing in 256K length long-form data.

### Conclusion

Qwen3-VL matched or exceeded Gemini 2.5 Pro and GPT-5 on several metrics -- the detailed benchmarks are worth checking in the paper directly.

As a side note, it has been about 3 weeks since the paper was released, and it has already been cited over 2,800 times, which speaks to how impressive the Qwen team's work truly is.
